# Política de Segurança

## Suporte de versões

Como o projeto está em evolução inicial, consideramos suportada a branch principal.

## Como reportar vulnerabilidades

Se você encontrar uma vulnerabilidade:

1. **Não** abra issue pública com detalhes sensíveis.
2. Envie relato privado com:
   - descrição do problema,
   - passos para reproduzir,
   - impacto potencial,
   - sugestão de mitigação (opcional).

## Escopo de segurança relevante

- Persistência local de dados.
- Dados de saúde e privacidade de usuário.
- Links externos e ações de telefone.

## Princípios

- Mínimo de dados necessários.
- Defesa em profundidade.
- Melhorias contínuas de privacidade e proteção.
